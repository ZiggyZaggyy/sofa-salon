# 测试「鸽王」与「徽章」

## 一、用管理员测试工具（最快）

1. 用**管理员账号**登录，进入 **Admin**。
2. 点击 **「测试鸽王与徽章」**（或 "Test 鸽王 & Badge"）。
3. 设置：
   - **鸽了次数 (0–3)**：用来预览血条和鸽子状态。  
     - 0 = 血条满三格  
     - 1/2 = 掉 1/2 格  
     - 3 = 变成鸽子（会显示鸽子图标）
   - **出席场次**：用来预览徽章。  
     - 3+ = 青铜 🥉  
     - 5+ = 白银 🥈  
     - 10+ = 黄金 🥇  
     - 20+ = 钻石 💎
4. 点 **「应用」** 保存。
5. 点 **「查看个人页」** 打开 `/profile`，即可看到：
   - 血条（上方三格）
   - 鸽子图标（当鸽了次数 = 3 时，头像下方会出现）
   - 徽章（当出席场次达到 3/5/10/20 时，头像下方会显示）
6. 想看**座位图上的血条**：去首页选一场活动 → 选座 → 进入该场次的座位图，你的座位上方会显示血条。

---

## 二、真实流程测试「鸽王」

- **掉血条 / 变鸽子**  
  - 方式 A：在「鸽王设置」里设好「开演前 X 小时」，在 X 小时内**取消自己的座位**，会算一次「鸽了」，血条掉一格；三次后变成鸽子。  
  - 方式 B：**管理员**在已结束场次的「出席记录」里，把某用户标成 **「鸽了」**，同样会掉一格；标三次即变鸽子。
- **从鸽子恢复**  
  管理员在同一场次把该用户标成 **「出席」**，连续两场都标出席后，鸽子状态会清除，血条归零。

---

## 三、真实流程测试「徽章」

- 徽章只看 **出席场次**（`attendance_count`）。
- 管理员在**已结束**的场次里，在「出席记录」中把某用户标成 **「出席」**，该用户的出席场次 +1。
- 同一用户被标满 3 / 5 / 10 / 20 场出席后，个人页和导航栏会显示对应徽章（青铜 → 白银 → 黄金 → 钻石）。

---

## 四、测试「挤一挤」

**重要**：`?testSqueeze=1` 必须加在**某场活动的座位图页面**的 URL 上，不是首页、个人页或 Admin 列表页。

1. **先进入座位图页面**（此时地址栏应为 `https://你的域名/screening/某串id`）  
   - **方式 A**：**首页** → 点某场活动 → 点「选座」→ 进入该场次的座位图。  
   - **方式 B**：**Admin** → 在活动列表里点某场活动的 **「选座」/「查看座位图」** 链接 → 会跳转到该场次的座位图。

2. **在地址栏当前 URL 末尾加上** `?testSqueeze=1`  
   - 例如：`https://你的域名/screening/abc123` → 改成 `https://你的域名/screening/abc123?testSqueeze=1`  
   - 回车刷新页面。

3. 刷新后应看到**挤一挤空位**（虚线框），点击后弹出「真的要挤进去吗？」→ 点「挤进去 ✦」完成预约。

4. **若点击「挤进去」后没反应**：现在页面上会显示具体错误信息（如「请先完善微信号」、座位已被占等），可根据提示排查。若仍异常，可打开浏览器开发者工具 (F12) 查看 Network 里 `/api/reserve` 的返回内容。

## 五、显示位置小结

| 功能     | 显示位置 |
|----------|----------|
| 血条     | 个人页顶部、座位图上「自己的」座位旁 |
| 鸽子图标 | 个人页头像下方（当 no_show_count ≥ 3） |
| 徽章     | 个人页头像下方、导航栏头像旁（当 attendance_count 达到 3/5/10/20） |
